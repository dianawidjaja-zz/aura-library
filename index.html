Build Scalable Web Apps

Aura is a component-based UI framework that provides intelligent client and server interaction.

View on Github Component Reference

Introduction

Components are the building blocks of Aura. With varying granularity, they encapsulate a modular reusable section of UI, providing comprehensive support for user interface events through component attributes and client-side controllers. Use these components out-of-the-box or extend and customize them based on your needs.

For a full list of component attributes and events, see the Reference site.

Alerts

ui:message

Display a message of varying severity level using the severity attribute. Set the closable attribute to true if displaying a closable button.

<ui:message title="Confirmation" severity="confirm">
    This is a confirmation message.
</ui:message>

<ui:message title="Information" severity="info" closable="true">
    This is a message.
</ui:message>

 <ui:message title="Warning" severity="warning">
    This is a warning.
</ui:message>

<ui:message title="Error" severity="error">
    This is an error message.
</ui:message>

Buttons

ui:button

Buttons can contain text only, image only, or both text and image. Setting labelClass="assistiveText" hides the label but maintains its accessibility.

<ui:button label="Submit"/>

<ui:button label="Search" class="img"/>
.THIS.uiButton.img {
    background:url(/img/icon/lightning16.png) no-repeat;
}

Input

HTML5 input types provide support for user interaction on different data types. For example, a field for numerical input displays a numerical keyboard on mobile devices. Each input field has an output equivalent for easy data manipulation and display. That means you can easily wire up the text value of a ui:inputText component in a ui:outputText component.

Autocomplete

ui:autocomplete

Autocomplete fields display suggestions as you type in a text or text area field. You can customize the handling of certain behaviors in a client-side controller, such as when a match is found, or when a result is selected.

<ui:autocomplete optionVar="row" 
        matchDone="{!c.handleInputChangeAutoCompleteMatchDone}"
        inputChange="{!c.handleInputChangeAutoComplete}" 
        selectListOption="{!c.handleSelectOptionAutoComplete}">
    <aura:set attribute="dataProvider">
        <auradocs:demoAutocompleteProvider/>
    </aura:set>
    <aura:set attribute="listOption">
        <ui:autocompleteOption label="{!row.label}" 
        keyword="{!row.keyword}" 
        value="{!row.value}" visible="{!row.visible}"/>
    </aura:set>
</ui:autocomplete>

Checkboxes

ui:inputCheckbox ui:outputCheckbox

Checkboxes are clickable and actionable, presented in a group for multiple selection. Just like the other input components, you can customize the checkbox behavior by handling the change event when its value changes. 

<ui:inputCheckbox value="true" label="select" change="{!c.doSomething}"/>

Radio Buttons

ui:inputRadio ui:outputRadio

A radio button can be grouped together for single selection.

<ui:inputRadio name="others" label="Prospecting" change="{!c.onGroup}"/>
<ui:inputRadio name="others" label="Qualification" change="{!c.onGroup}" value="true"/>
<ui:inputRadio name="others" label="Needs Analysis" change="{!c.onGroup}"/>
<ui:inputRadio name="others" label="Closed Lost" change="{!c.onGroup}"/>
<p>Selected Items:</p>
<p><ui:outputText class="result" aura:id="radioGroupResult" value="" /></p>
onGroup: function(cmp, evt) {
     var elem = evt.getSource().getElement();
     var selected = elem.textContent;
     resultCmp = cmp.find("radioGroupResult");
     resultCmp.set("v.value", selected);
}

Dialogs

ui:panelDialog ui:panelManager

A modal dialog disables the background until the user interacts with it. A non-modal dialog is closed when the user clicks anywhere outside of it. Embed an instance of ui:dialogManager within your component to manage your ui:panelDialog instances. Fire the force:showPanel and force:hidePanel events with the dialog you want to open and any other appropriate attributes. Firing force:showPanel creates an instance of ui:panelDialog.

<ui:panelManager />
showModal: function(cmp, event, helper) {
    var cmps = $A.componentService
              .newComponentAsync(
                  this, function(){}, helper.getConfig(cmp)
               );
    var overlay = cmps[3];
    overlay.addHandler('press', cmp, 'c.showModal')
    var openPanelEvent = $A.get('e.ui:openPanel').setParams({
          show: true,
            isModal: true,
            isDialog: true,
          title: 'my panel',
          'class': 'myClass',
          body: cmps
        });
        openPanelEvent.fire();
  }

Layouts

Using layouts is a quick way to position your components in a horizontal or vertical grid.

Horizontal

ui:block

ui:block provides a horizontal layout for your components. It's rendered in div tags in the order: bLeft, bRight, and bBody. Visually, the body appears between the left and right facets.

<ui:block>
    <aura:set attribute="left">
        <ui:image src="/../img/search.png" alt="bLeft" />
    </aura:set>
    <aura:set attribute="right">
        <ui:button label="Submit"/>
    </aura:set>
    <ui:inputText label="Text" labelPosition="hidden" /> 
</ui:block>

Vertical

ui:vbox

ui:vbox creates a vertical box area on a page with simple wrapper HTML. Insert content into the top and bottom area by adding it to the north and south facets respectively. The body is inserted in the middle of top and bottom area. 

<ui:vbox>
    <aura:set attribute="north">
        This <code>north</code> facet is rendered in a div element.
    </aura:set>
    <aura:set attribute="south">
        This <code>south</code> facet is rendered in a div element.
    </aura:set>
    <p>The body is nested in the middle of the north and south facets.</p>
</ui:vbox>

Lists

aura:iteration

aura:iteration iterates over a collection of items and renders the body of the tag for each item. Data changes in the collection are rerendered automatically on the page. It also supports iterations containing components that are created exlusively on the client-side or components that have server-side dependencies.

<aura:iteration items="1,2,3,4,5" var="item">
    <meter value="{!item / 5}"/>
</aura:iteration>

Menus

ui:menu ui:menuList ui:actionMenuItem ui:checkboxMenuItem ui:radioMenuItem ui:menuItemSeparator

A dropdown menu contains a trigger and list items. The list items can be wired up to a client-side controller to trigger a desired action. ui:menu extends ui:popup. 

<ui:menu>
    <ui:menuTriggerLink aura:id="mytrigger" label="Select status"/>
    <ui:menuList>
        <ui:actionMenuItem label="Any" click="{!c.doSomething}"/>
        <ui:actionMenuItem label="Open" click="{!c.doSomething}"/>
        <ui:actionMenuItem label="Closed" click="{!c.doSomething}"/>
        <ui:actionMenuItem label="Closed Won" click="{!c.doSomething}"/>
        <ui:menuItemSeparator/>
        <ui:checkboxMenuItem label="Red"/>
        <ui:checkboxMenuItem label="Blue"/>
        <ui:checkboxMenuItem label="Green"/>
        <ui:checkboxMenuItem label="Yellow"/>
        <ui:menuItemSeparator/>
        <ui:radioMenuItem label="San Francisco"/>
        <ui:radioMenuItem label="New York"/>
        <ui:radioMenuItem label="Los Angeles"/>
        <ui:radioMenuItem label="San Diego"/>
        <ui:radioMenuItem label="Seattle"/>
    </ui:menuList>
</ui:menu>

Popups

ui:popup

A popup has a trigger that controls its visibility. Setting curtain="true" creates an overlay under the popup. Setting attachToBody="true" attaches the popup to the HTML body tag level.

<ui:popup label="trigger">
    <ui:popupTrigger>Click this trigger to open popup</ui:popupTrigger>
    <ui:popupTarget>
        This displays in a popup.
    </ui:popupTarget>
</ui:popup>

Spinners

ui:spinner

A spinner indicates that content is being loaded. It can be displayed or hidden accordingly.

 <ui:spinner aura:id="spinner"/>
({
    toggleSpinner: function(cmp, event) {
        var spinner = cmp.find('spinner');
        var evt = spinner.get("e.toggle");
        
    if(!$A.util.hasClass(spinner.getElement(), 'hideEl')){
        evt.setParams({ isVisible : false });
     }    
    else {
        evt.setParams({ isVisible : true });
  }
    evt.fire();
    }
})
